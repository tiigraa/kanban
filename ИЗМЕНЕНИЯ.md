## Как это работает вместе:

1. Пользователь создает задачу на странице "Панель администратора"
2. Задача валидируется и сохраняется в localStorage
3. Пользователь автоматически перенаправляется на страницу "Активный спринт"
4. При загрузке страница спринта читает задачи из localStorage
5. Новые задачи добавляются в колонку "К выполнению"
6. Пользователь может:
   - Фильтровать задачи по исполнителю
   - Перетаскивать задачи между колонками
   - Видеть актуальное количество задач в каждой колонке

## Технические детали:

### Фильтрация:
- Реализована через функцию высшего порядка `filterTasks()`
- Использует `useMemo` для оптимизации
- Работает реактивно при изменении фильтра

### Перетаскивание:
- Использует нативный HTML5 Drag and Drop API
- Состояние перетаскивания хранится в `draggedTask` и `draggedFrom`
- При сбросе задачи обновляется состояние `columns`

### Сохранение:
- Задачи хранятся в localStorage как JSON
- При создании задаче присваивается уникальный ID (временная метка)
- После загрузки задач localStorage очищается (чтобы избежать дублирования)

## Запуск проекта:

1. Установите зависимости (если еще не установлены):
   ```
   npm install
   ```

2. Запустите проект:
   ```
   npm start
   ```

3. Откройте в браузере: http://localhost:3000

## Проверка работы:

- **Фильтр**: выберите пользователя в выпадающем списке
- **Перетаскивание**: зажмите карточку мышью и перетащите в другую колонку
- **Создание**: заполните форму и нажмите "Создать задачу"
